# 诊断报告反馈机制设计 v1

> 目的：让运营（或其他看报告的人）能结构化地反馈"哪句对、哪句不对、为什么"，帮助小q修正诊断认知。

---

## 一、问题

现在的流程是：小q出报告 → Seth看 → 口头说"这里不对" → 小q改。

问题：
1. **只有Seth能反馈**——如果运营看报告，他们的反馈无法结构化回传
2. **反馈粒度粗**——"菜单那块比较薄弱"，但具体是哪句话、哪个判断有问题？
3. **反馈容易丢失**——口头讨论完如果没记录，下次还会犯同样的错

## 二、设计方案

### 2.1 报告格式改进

**每个诊断结论标编号。** 不是无序的段落，而是带编号的判断列表。这样反馈时可以精确引用。

示例：
```
### 1.3 深层产品问题

[P1] 核心产品定义模糊——店名推黑椒土豆泥(月售13)，实际热卖是酸菜(116)和烧椒(99)。
[P2] 两个口味只有量的升级没有内容的升级——缺少"主体+配角+小菜"的套菜组合。
[P3] 58%加肉卷+57%加卤蛋=大多数人的标配没被设计成产品。
...

### 2.6 满减分析

[A1] 满38减1折扣率2.6%，在列表页是负面展示。
[A2] 门槛¥38和客单¥27-30之间差¥11，错位。
[A3] 满返门槛¥58 vs 客单¥30，完全触发不了。
```

### 2.2 反馈表

随每份报告附一份反馈表，格式：

| 编号 | 小q的判断 | 对/不对/部分对 | 你的看法（为什么/正确的应该是什么） | 优先级(高/中/低) |
|------|---------|-------------|-------------------------------|---------------|
| P1 | 核心产品定义模糊 | 对 | — | — |
| P2 | 缺套菜组合 | 部分对 | 套菜方向对但不一定要配小菜，看品类 | 高 |
| A1 | 满38减1是负面展示 | 不对 | 这个商圈满减本来就低，不算减分 | 高 |

**反馈表可以是：**
- Excel/在线表格（运营直接编辑）
- 微信消息（复制编号+反馈内容）
- 纸质打印后手写（线下过的时候）

### 2.3 反馈回传格式

运营或Seth填完反馈表后，回传给小q的标准格式：

```
店铺：酸菜猪脚饭（鼓楼店）
日期：2026-04-15
反馈人：运营张三 / Seth

## 对的（确认）
P1, P3, P5, A2, A3

## 不对的（纠正）
P2: 不是缺套菜，是缺什么什么...（正确理解）
A1: 这个商圈满减力度本来就低，38-1不算减分

## 部分对的（补充）
P4: 方向对但不是所有品类都适用，猪脚饭适用是因为...

## 采纳的建议
建议1: 推广起量 — 采纳，已充值¥300
建议4: 菜单重设计 — 部分采纳，只做酸菜系列先测
建议5: 改满减 — 不采纳，先观察

## 不采纳的原因
建议5不采纳：现在改满减对销量影响不大，等菜单改完再动
```

### 2.4 反馈入口

为了让运营也能方便反馈（不只是Seth）：

**方案A：在线文档**
- 每份报告生成一个在线反馈表（腾讯文档/飞书文档）
- 运营打开链接直接编辑
- 小q定期拉取反馈

**方案B：微信模板消息**
- 报告发给运营后，附一条模板消息：
  ```
  看完报告了吗？帮我看下这几个判断对不对：
  P1: 核心产品是酸菜和烧椒 → 对/不对？
  P2: 缺套菜组合 → 对/不对？
  A1: 满减38-1太低 → 对/不对？
  有什么想补充的？
  ```
- 运营回复"P1对 P2不对 因为xxx"

**方案C：PDF里嵌入二维码**
- 报告PDF底部放一个反馈表的二维码
- 扫码进入在线表格填写

先用方案B起步（最轻量），后续做产品化再用A/C。

---

## 三、关键原则

1. **反馈要精确到"哪句话"**——用编号，不要笼统说"菜单部分不好"
2. **"不对"比"对"更有价值**——纠正错误是学习的核心
3. **不采纳的原因比采纳更重要**——为什么不做，往往暴露了小q的认知盲区
4. **每条纠正都要说"为什么"**——不只是"不对"，要说正确的是什么、为什么
5. **反馈门槛要低**——运营不会填复杂的表，3分钟能完成最好
